Hoisting

Conceptually, a strict definition of hoisting means something is moved to the top. but this is not the case in Javascript. What It actually does is that Javascript puts its variable declaration and funtion declaration into memory during compilaton but stay exactly where it is written in the code.

One of the advantages of Javascript putting its functiona and vaiable declarations into memeory is that it allows you to use the variable and the function before declaring it in your code.

function catName(name) {

console.log("My cat's name is " + name);

}

catName("Tiger");

/* The result of the code above is: "My cat's name is Tiger" */

The above code snippet is probably the way we will write our code to make it work. but what if we call the function before its is declared.

catName("Chloe");

function catName(name) {

console.log("My cat's name is " + name);

}

/* The result of the code above is: "My cat's name is Chloe" */

Even though we call the function in our code first before the function is written, the code still works. This is because of Hoisting.
