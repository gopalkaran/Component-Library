Closure

Before we talk about closure we need to know about scope. Scope is a very important concept that manages the availabilty of the variables, functions and objects in some particular part of your code. In other words, scope determines visibility of the varibles and functions in areas of your code.

So the point of restricting the visibility of the variables and not allowing its availabilty everywhere is that scope provides some level of security. Users should have accesed to only that resource which is necessary for them. Suppose you have a computer which is accessed by many users other than you. lets say you are the administrator, you have full access to the system files, you can read or modify drivers of the system. you should not give access to use the administrator account to other users. They should be given the Guest account so that they can only make changes to a particular things or programs. This way you can save your program or systems from unauthorized access.

Scope also solves the naming problems when we have variables with the same name but in different scopes.

lets understand scopes by a simple example:

const name="Gopal";

console.log(name); //Gopal

Here We can easily log this variable. but now if we move the variable declaration inside if block.

if (true) {

const name = 'Gopal';

}

console.log(name); // ReferenceError: name is not defined

This time when we are trying to log the variable. It gives an ReferenceError : name is not defined. The name varible has scope only within the if block means it is only available inside if block. we cannot acces it outside the if block.

There are serveral type of scopes in javascript

  • Block Scope
  • Function Scope
  • Module Scope
  • Global Scope
  • Lexical Scope

Block Scope

A code block in javascript creates a scope for variables declared using let and const.

if (true) {

// "if" block scope

const name = 'Gopal';

console.log(name); // 'Gopal'

}

console.log(name); // throws ReferenceError

The first console.log(name) correctly logs the variable 'name' because the name is accesed within its scope.

The second console.log(name) throws an error because we are trying to access the variable outside the if block. Const is block scoped.

var is not block scoped

if (true) {

// "if" block scope

var name = "Gopal";

console.log(name); // "Gopal"

}

console.log(name); // "Gopal"

Function Scope

Function in javascript creates scope for variables declared using var, let, const.

function run() {

// "run" function scope

var name = 'Gopal';

console.log(name); // 'Gopal'

}

run();

console.log(name); // throws ReferenceError

The run function creates a scope for the variable name. The variable name is accesible inside the function but inaccessible outside the function. Same works for let and const.

Module Scope

Module also creates scope for variables, functions and classes.

Let's say a module sayHi.js alerts 'Hello' to the user.

// sayHi.js

function sayHi(user){

alert(`Hello, ${user}!`)

}

Another module main.js imports sayHi.js module and uses it.

// main.js

import {sayHi} from './sayHi.js';

sayHi('Gopal'); // throws ReferenceError

function sayHi is not accessible outside sayHi module unless explicitly exported using export keyword.

// sayHi.js

export function sayHi(user){

alert(`Hello, ${user}!`)

}

Now we can access the sayHi function from another module, simply importing it like the main.js module written above.

Global Scope

If a variable is declared outside all the functions or outside blocks, it is said to be defined in the global scope. Variables in outer scope is accessible from any local scope. A varible declared inside global scope is known as global variable.

const name = 'Gopal'

function sayHello () {

console.log(name)

}

console.log(name) // 'Gopal'

sayHello() // 'Gopal'

Although you can declare variable in global scope but it is not advised. Declaring variables in global scope will create naming collisions when two or more variable have same name.

let name = 'Gopal'

let name = 'Gopal Karan' // Error

Variables declared using let or const keyword will give an error whenever name collision happens.

var name = 'Gopal'

var name = 'Gopal Karan' // will overwrite 'Gopal'

Variables declared using var keyword will overwrite the previous value whenever name collision happens.

Lexical Scope

When one function is defined inside another function, inner function can access variables of outer function. This is called nested scope or lexical scope.

function outer () {

const outerVar = `I'm the outer variable`

function inner() {

const innerVar = `I'm the inner variable`

console.log(outerVar) // I'm the outer variable

}

console.log(innerVar) // Error, innerVar is not defined

}

The lexical scope consists of outer scopes determined statically. In this example, the lexical scope of inner() function consists of the scope of outer() function.
